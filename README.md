# Frontend Code Study

## Vue3 企业 ERP 上手计划

目标：**2.5 个月内，从 0 到能上手企业 ERP 前端项目**。

这份计划不是泛泛学前端，而是围绕一条真实业务主线推进：**登录、权限、菜单、列表、表单、CRUD、接口联调、TypeScript、工程化**。70 天后，你至少要具备“接到一个后台管理需求，能独立拆解并做出来”的能力。

学习强度假设：**每天 6–8 小时**。如果某天时间不足，优先完成当天的**产出**和**验收**，不要只停留在“看懂了”。

## 这份 README 适合谁

- 想在较短时间内补齐 **Vue3 + TS + 后台管理系统开发能力** 的同学
- 目标明确，想尽快达到“能入职、能接活、能上手 ERP 项目”的同学

## 核心主线项目（贯穿全程）

项目名建议：`erp-admin`

技术栈：**Vue3 + Vite + TypeScript + Vue Router + Pinia + Element Plus + Axios**

你最终要做出来的能力闭环：

- **登录**：token 登录态、退出、自动续登（本地存储）
- **权限**：角色/菜单/按钮权限（至少做到菜单权限 + 路由守卫）
- **通用列表页**：查询表单 + 分页 + 排序 + loading/空态
- **通用表单页**：校验 + 回显 + 提交 + 联动（下拉/树形选择）
- **弹窗/抽屉 CRUD**：新增/编辑复用同一套表单
- **接口封装**：统一 request、错误处理、提示、文件下载/上传
- **工程化**：ESLint/Prettier、环境变量、Git 工作流、调试能力

> 备注：没有后端也能做。前期用本地 mock（或 JSON Server/简单 mock 文件）跑通流程；后期再替换为真实接口更容易上手。

### 建议目录结构

如果你准备把这份计划真正落地成一个项目，建议尽早按后台项目的思路组织目录：

```text
erp-admin/
  src/
    api/
    components/
    composables/
    layout/
    router/
    stores/
    utils/
    views/
  mock/
  docs/
```

这样做的好处是：后面学到接口封装、通用表格、登录态、动态路由时，不需要反复推倒重来。

## 10 周路线总览

先记住阶段目标，再去执行每天的细项：

- **第 1 周**：只求把后台常见页面静态结构做顺，不追求花哨效果
- **第 2–3 周**：补齐 JavaScript 数据处理与异步能力，能处理接口数据
- **第 4 周**：进入 Vue3，学会组件拆分、响应式、基础页面开发
- **第 5 周**：完成登录态、路由守卫、菜单与动态路由
- **第 6 周**：进入企业日常开发核心，完成接口封装和 CRUD 闭环
- **第 7 周**：把 TypeScript 真正接进项目，而不是停留在语法层
- **第 8–9 周**：补齐上传下载、字典、按钮权限、通用弹窗、复杂表单联动
- **第 10 周**：做交付收尾，补规范、调试、表达和演示能力

你可以把 10 周拆成 4 个里程碑来看：

- **里程碑 1**：能独立还原后台静态页面，并能用 JS 处理常见数据
- **里程碑 2**：能用 Vue3 搭出带登录和菜单的后台骨架
- **里程碑 3**：能完成至少 2 个真实 CRUD 页面并接好接口
- **里程碑 4**：能把项目讲清楚、调通、演示，并具备入职上手能力

## 10 周（70 天）企业专家级每日计划

说明：

- 每一天都包含：**目标**、**产出**、**验收**（完成验收才算过关）
- 如果你想更“像上班”：每天结束前把当天产出提交到 Git（哪怕只是一点点）

### 第 1 周：HTML/CSS 够用即可（目标：能还原 ERP 页面框架）

**Day 1**
- **目标**：搞懂 HTML 基础结构与常见标签用途
- **产出**：1 个“登录页”静态 HTML（含表单）
- **验收**：能解释 `form/input/label/button` 各自用途；页面在浏览器可用

**Day 2**
- **目标**：盒模型、margin/padding/border、常见单位
- **产出**：把登录页做出“居中卡片 + 标题 + 输入框间距”
- **验收**：能说清 `box-sizing` 的差别；不同屏幕宽度不炸版

**Day 3**
- **目标**：Flex 布局（ERP 使用频率极高）
- **产出**：做一个“顶部栏 + 侧边栏 + 内容区”的布局骨架
- **验收**：侧边栏固定宽度、内容区自适应；窗口缩放布局仍合理

**Day 4**
- **目标**：常见表单控件样式与状态（hover/focus/disabled）
- **产出**：做一个“新增用户”表单静态页（含必填标识）
- **验收**：focus 状态明显；表单分组对齐整齐

**Day 5**
- **目标**：常见排版（字体、行高、颜色、间距）与可读性
- **产出**：做一个“列表页”静态（查询区 + 表格样式 + 分页样式）
- **验收**：查询区与表格层级清晰；表格行 hover 有反馈

**Day 6**
- **目标**：常用 CSS 选择器与优先级（够用即可）
- **产出**：把 Day3 的布局骨架做成更像后台（间距/阴影/颜色）
- **验收**：能解释“为什么这个样式生效/不生效”（优先级/覆盖）

**Day 7**
- **目标**：复盘 + 小测
- **产出**：把 3 个静态页面（登录/列表/表单）整理到一个文件夹
- **验收**：能在 10 分钟内从零写出一个“后台三栏布局”

### 第 2–3 周：JavaScript 核心（目标：接口数据处理 + 异步能力）

**Day 8**
- **目标**：变量、类型、基本语法；写出可读的函数
- **产出**：写 10 个小函数（字符串/数组/对象基础处理）
- **验收**：每个函数都有输入输出示例；能跑通

**Day 9**
- **目标**：数组对象常用方法（map/filter/reduce/sort/find）
- **产出**：实现“列表数据加工”：把接口原始数据变成表格要展示的字段
- **验收**：能说明为何用 map/filter；不写多层 for 也能完成

**Day 10**
- **目标**：对象操作、解构、可选链、空值处理（ERP 常见脏数据）
- **产出**：实现“表单回显数据转换”（后端字段 → 前端表单模型）
- **验收**：遇到 null/undefined 不崩；字段缺失有默认值策略

**Day 11**
- **目标**：函数作用域、闭包（理解即可，避免踩坑）
- **产出**：写 3 个闭包练习（计数器、一次性函数、私有变量）
- **验收**：能解释“闭包为什么能记住变量”

**Day 12**
- **目标**：this / 原型基础（够用即可，重点是别被坑）
- **产出**：写 5 个 this 场景例子（普通函数/箭头函数/对象方法）
- **验收**：能预测输出；知道在 Vue 里为何更推荐组合式 API

**Day 13**
- **目标**：事件循环、宏任务/微任务（理解 Promise 顺序）
- **产出**：写 5 个 Promise 输出顺序题并解释
- **验收**：能讲清“为什么 then 先/后执行”

**Day 14**
- **目标**：复盘 + 代码规范（命名/拆函数/早返回）
- **产出**：把 Day8–13 的练习整理到 `Javascript/` 下并归类
- **验收**：任何一个练习你都能在 2 分钟内解释清楚

**Day 15**
- **目标**：Promise 基础与链式调用
- **产出**：封装一个“模拟请求”的 Promise（setTimeout 模拟）
- **验收**：能处理成功/失败；能链式处理结果

**Day 16**
- **目标**：async/await + try/catch（项目里主要用它）
- **产出**：用 async/await 改写 Day15；加上错误提示策略
- **验收**：try/catch 覆盖完整；出错不会静默失败

**Day 17**
- **目标**：并发（Promise.all / allSettled）与依赖请求
- **产出**：模拟“先拿用户信息，再拿菜单”的两段请求流程
- **验收**：知道何时并发、何时串行；能写出清晰代码

**Day 18**
- **目标**：防抖/节流（搜索框、窗口 resize 常用）
- **产出**：手写 debounce/throttle（能用即可）
- **验收**：能在输入框 demo 里看到效果差异

**Day 19**
- **目标**：模块化与组织代码（import/export 思维）
- **产出**：把“数据转换函数”拆为模块（例如 `formatters`）
- **验收**：入口文件简洁；复用函数能被多个文件引用

**Day 20**
- **目标**：复盘 + 小实战
- **产出**：给你一份 JSON（自造即可），生成“树形菜单数据 + 面包屑”
- **验收**：树结构正确；能处理多级 children

**Day 21**
- **目标**：阶段验收
- **产出**：做一个纯 JS 小项目：控制台版“用户管理”（增删改查数据）
- **验收**：功能完整；代码可读；能解释数据结构设计

## 你接下来怎么用这份计划

- 每天只盯住当天的 **目标 / 产出 / 验收**，不要提前焦虑后面的内容
- 每周结束时回看一次「路线总览」，确认自己有没有偏离主线项目
- 做完就把对应代码放进仓库，最好保持持续提交，而不是堆到最后一起补
- 主线项目建议单独放一个文件夹，练习代码和项目代码尽量分开
- 复盘时只写 3 件事：**今天学到的 1 个关键点、踩的 1 个坑、明天的 1 个改进**
- 如果某一天掉队，不要补“学习时长”，优先补齐“可运行产出”

## 一句话执行原则

**先做出能跑的结果，再去补理解；先完成主线项目，再去扩展细节。**
